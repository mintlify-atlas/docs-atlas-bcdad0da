## GET /users/me

Retrieves the current user's profile information, including roles and permissions. Requires authentication.

Alias: `GET /profile` (for client compatibility)

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login. Format: `Bearer <token>`
</ParamField>

### Response

<ResponseField name="id" type="integer">
  User's unique identifier.
</ResponseField>

<ResponseField name="email" type="string">
  User's email address.
</ResponseField>

<ResponseField name="username" type="string | null">
  User's username, or null if not set.
</ResponseField>

<ResponseField name="displayName" type="string">
  Display name for the user. Uses username if available, otherwise falls back to email.
</ResponseField>

<ResponseField name="roles" type="string[]">
  Array of roles assigned to the user. Possible values: `"admin"`, `"user"`
</ResponseField>

<ResponseField name="permissions" type="string[]">
  Array of permissions granted to the user. Examples:
  - `"books:read"` - All users
  - `"books:download"` - Admin only
  - `"books:upload"` - Admin only
  - `"books:delete"` - Admin only
</ResponseField>

<ResponseField name="created" type="string">
  ISO 8601 timestamp of when the user account was created.
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:3000/users/me \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

### Example Response (Admin User)

```json
{
  "id": 1,
  "email": "admin@example.com",
  "username": "admin",
  "displayName": "admin",
  "roles": ["admin", "user"],
  "permissions": [
    "books:read",
    "books:download",
    "books:upload",
    "books:delete"
  ],
  "created": "2026-03-04T10:30:00Z"
}
```

### Example Response (Regular User)

```json
{
  "id": 2,
  "email": "user@example.com",
  "username": "johndoe",
  "displayName": "johndoe",
  "roles": ["user"],
  "permissions": ["books:read"],
  "created": "2026-03-04T11:45:00Z"
}
```

### Status Codes

- `200` - Successfully retrieved user profile
- `401` - Missing or invalid authentication token
- `404` - User not found in database
- `500` - Server error retrieving user details

### Notes

- This endpoint requires a valid JWT token in the Authorization header
- The first user created is automatically assigned admin role
- Admin users have additional permissions for uploading, downloading, and deleting books
- The `/profile` endpoint is an alias and returns identical data
