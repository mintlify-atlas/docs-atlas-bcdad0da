## List Archived Books

```
GET /books/archive
```

Retrieve all books that have been archived.

### Authentication

No authentication required.

### Response

<ResponseField name="books" type="array">
  Array of archived book objects
  
  <ResponseField name="id" type="integer">
    Unique identifier for the book
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Book title
  </ResponseField>
  
  <ResponseField name="author" type="object">
    Author information
    
    <ResponseField name="id" type="integer">
      Author's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Author's name
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="status" type="string">
    Book status ("archived" for this endpoint)
  </ResponseField>
  
  <ResponseField name="archived_at" type="string" optional>
    ISO 8601 timestamp when the book was archived
  </ResponseField>
  
  <ResponseField name="archive_reason" type="string" optional>
    Reason provided for archiving the book
  </ResponseField>
</ResponseField>

### Example Request

```bash
curl https://your-dust-server.com/books/archive
```

### Example Response

```json
{
  "books": [
    {
      "id": 42,
      "name": "Old Book Title",
      "author": {
        "id": 12,
        "name": "Author Name"
      },
      "status": "archived",
      "archived_at": "2024-02-10T15:45:00Z",
      "archive_reason": "Duplicate copy"
    }
  ]
}
```

## Archive a Book

```
POST /books/:id/archive
```

Archive a book, removing it from the active library while preserving its data.

### Authentication

**Required.** Must include a valid JWT token in the Authorization header.

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the book to archive
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

<ParamField header="Content-Type" type="string" optional>
  Must be `application/json` if providing a request body
</ParamField>

### Request Body (Optional)

<ParamField body="reason" type="string" optional>
  Reason for archiving the book
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

### Example Request (With Reason)

```bash
curl -X POST \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{"reason": "Duplicate copy"}' \
  https://your-dust-server.com/books/42/archive
```

### Example Request (Without Reason)

```bash
curl -X POST \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  https://your-dust-server.com/books/42/archive
```

### Example Response

```json
{
  "message": "Book archived successfully"
}
```

### Error Responses

#### 400 Bad Request

```json
{
  "error": "Invalid book ID"
}
```

```json
{
  "error": "Invalid JSON"
}
```

#### 401 Unauthorized

```json
{
  "error": "Unauthorized"
}
```

## Unarchive a Book

```
DELETE /books/:id/archive
```

Restore an archived book to the active library.

### Authentication

**Required.** Must include a valid JWT token in the Authorization header.

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the book to unarchive
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

### Example Request

```bash
curl -X DELETE \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  https://your-dust-server.com/books/42/archive
```

### Example Response

```json
{
  "message": "Book unarchived successfully"
}
```

### Error Responses

#### 400 Bad Request

```json
{
  "error": "Invalid book ID"
}
```

#### 401 Unauthorized

```json
{
  "error": "Unauthorized"
}
```

### Notes

- Archiving a book sets its status to "archived" and records the timestamp
- Archived books are excluded from the main `/books` listing
- Unarchiving restores the book status to "active" and clears the archive timestamp and reason
- All book data (metadata, tags, reading progress) is preserved when archiving
