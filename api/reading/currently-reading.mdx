## Endpoint

```
GET /reading/currently-reading
```

Returns a list of books that the authenticated user is currently reading (progress between 0% and 100%, exclusive).

## Authentication

This endpoint requires authentication. Include your API token in the request headers. The endpoint returns reading progress specific to the authenticated user.

## Request

No parameters required.

### Example Request

```bash
curl -X GET https://your-dust-server.com/reading/currently-reading \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Response

Returns an object containing an array of books with reading progress information. Books are ordered by `last_read_at` in descending order (most recently read first).

## Implementation Details

The endpoint filters reading progress records where:
- `user_id` matches the authenticated user
- `percentage_complete > 0 AND percentage_complete < 1`

Results are ordered by `last_read_at` in descending order.

## Status Codes

- `200` - Success
- `401` - Unauthorized (missing or invalid authentication)
- `500` - Internal server error
