## Get User

Retrieve details for a specific user.

### Endpoint

```
GET /admin/users/:id
```

### Path Parameters

<ParamField path="id" type="integer" required>
  The user's unique identifier
</ParamField>

### Response

<ResponseField name="user" type="object">
  User object containing user details
  
  <ResponseField name="id" type="integer">
    Unique user identifier
  </ResponseField>
  
  <ResponseField name="username" type="string | null">
    Username (optional)
  </ResponseField>
  
  <ResponseField name="email" type="string">
    User's email address
  </ResponseField>
  
  <ResponseField name="is_admin" type="boolean">
    Whether the user has admin privileges
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of user creation
  </ResponseField>
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:3000/admin/users/2 \
  -H "Authorization: Bearer YOUR_ADMIN_JWT_TOKEN"
```

### Example Response

```json
{
  "user": {
    "id": 2,
    "username": "john_doe",
    "email": "john@example.com",
    "is_admin": false,
    "created_at": "2024-01-16T14:22:00Z"
  }
}
```

## Update User

Update one or more fields for a specific user.

### Endpoint

```
PUT /admin/users/:id
```

### Path Parameters

<ParamField path="id" type="integer" required>
  The user's unique identifier
</ParamField>

### Request Body

All fields are optional, but at least one must be provided.

<ParamField body="username" type="string">
  New username for the user
</ParamField>

<ParamField body="email" type="string">
  New email address for the user
</ParamField>

<ParamField body="is_admin" type="boolean">
  Update admin privileges
</ParamField>

### Response

Returns the updated user object.

<ResponseField name="user" type="object">
  Updated user object
</ResponseField>

### Example Request

```bash
curl -X PUT http://localhost:3000/admin/users/2 \
  -H "Authorization: Bearer YOUR_ADMIN_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "jane_doe",
    "is_admin": true
  }'
```

### Example Response

```json
{
  "user": {
    "id": 2,
    "username": "jane_doe",
    "email": "john@example.com",
    "is_admin": true,
    "created_at": "2024-01-16T14:22:00Z"
  }
}
```

## Delete User

Permanently delete a user from the system.

### Endpoint

```
DELETE /admin/users/:id
```

### Path Parameters

<ParamField path="id" type="integer" required>
  The user's unique identifier
</ParamField>

### Response

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

### Example Request

```bash
curl -X DELETE http://localhost:3000/admin/users/2 \
  -H "Authorization: Bearer YOUR_ADMIN_JWT_TOKEN"
```

### Example Response

```json
{
  "message": "User deleted successfully"
}
```

### Error Responses

<ResponseField name="400 Bad Request">
  Cannot delete your own account
</ResponseField>

<ResponseField name="404 Not Found">
  User not found
</ResponseField>

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
</ResponseField>

<ResponseField name="403 Forbidden">
  User does not have admin privileges
</ResponseField>
