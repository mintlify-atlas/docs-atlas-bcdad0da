Dust is a high-performance media server built with Zig, designed specifically for ebook and comic book enthusiasts. It provides automatic metadata fetching, intelligent folder organization, and a beautiful web interface for managing your digital library.

## Why Dust?

Dust combines the power of a modern media server with specialized features for book lovers:

- **Self-Hosted**: Complete control over your data and library
- **Fast & Efficient**: Built with Zig for exceptional performance and low resource usage
- **Automatic Metadata**: Fetches book information from OpenLibrary and Google Books API
- **Smart Organization**: Handles both ebooks (EPUB, PDF) and comics with folder-based structure
- **Web Interface Built-In**: No separate client installation needed
- **Docker & Native**: Deploy however you prefer

## Architecture

Dust is built with modern technologies for reliability and performance:

- **Backend**: Zig 0.15.2 with http.zig framework
- **Database**: SQLite (bundled with binary)
- **Authentication**: JWT-based with role-based permissions
- **Background Tasks**: Automatic library scanning and cleanup
- **Frontend**: Modern web client served directly by the server

## Quick Example

Get started with Dust in seconds:

```bash
# One-line installation on Ubuntu/Debian
curl -fsSL https://raw.githubusercontent.com/dust-books/dust-server/main/scripts/install.sh | sudo bash

# Or with Docker
docker run -d \
  -p 4001:4001 \
  -v /path/to/books:/app/books:ro \
  -v dust_data:/app/data \
  -e JWT_SECRET=$(openssl rand -base64 32) \
  -e DUST_DIRS=/app/books \
  dust-server:latest
```

Then open `http://localhost:4001` in your browser to create your admin account and start organizing your library.

## System Requirements

<Note>
  Dust is designed to be lightweight and can run on modest hardware:
  - **CPU**: Any modern x86_64 or ARM64 processor
  - **RAM**: 256MB minimum (512MB recommended)
  - **Storage**: Depends on your library size
  - **OS**: Linux (Ubuntu/Debian recommended), Docker, or build from source
</Note>
